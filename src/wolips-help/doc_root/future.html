Release Plan

WOLips development is driven by bug reports and feature requests posted at SourceForge. Bugs that are not reported won't get fixed, and features that aren't requested will not materialise.

WOLips is a collaborative open source effort. Please consider contributing!

There are three kinds of relases:

  1. Bug Fix - increments z in x.y.z - with no new features.
  2. Feature - increments y in x.y.z - new features, and possibly some bugs fixed.
  3. Major - increments x in x.y.z - with major additions and/or modifications.

Tracker items (bug reports and feature requests) have a Status, a Category and a Group. The Group indicates which release will address the item, and the Category which part of WOProject.

The version number digits, x.y.z, have meaning:

  • Incrementing 'z' is done for bug fix releases. These have no new features.
  • Incrementing 'y' is done for feature releases. These have new features, and possibly some bugs fixed.
  • Incrementing 'x' is done for feature releases
