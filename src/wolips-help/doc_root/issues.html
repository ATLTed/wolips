Bugs

Bug

Fix
Importing projects with localized components doesn't work. ?

When you start eclipse for the first time with WOLips installed a file ~\Library\wobuild.properties should be created in your user home directory. Make sure this actually did happen!

If not you’ll have to create it yourself. It should contain this:

#This file stores the wo specific properties
#Mon Nov 18 16:07:32CET 2002
wo.wolocalroot=C:\\Apple\\Local
wo.woroot=C:\\Apple

Take note of the double back-slashes “\\”.
Importing projects with java code organised in packages doesn't work properly. Either the java files are not found or they may be put in the wrong directory. Adding them manually is not a big task!
When you add a jar file (library, corresponding to framework cuurently under development in eclipse) to a project's java build path it is not included in the classpath of the built application. Install a copy of the framework in question, and do 'Add Local Framework'.
Doing 'Add Local Framework' gives different results (in .classpath) on Windows and Macintosh - none of which works when you move the project to the other platform. ?
When building on Macintosh the CLSSPATH.TXT (for Windows deployemnt) file frequently contains path like this: *.framewor\Versions\A\Resources\Java\*.jar Build on Windows or clean up CLSSPATH.TXT manually.

Issue

Fix
Currently you can not use the ant scripts outside of Eclipse.  
The import (project) Wizards currently only work with PBWO projects. If you try it with PBX projects you may find that some file where skipped and that the eclipse .project file in incomplete.

How to import an existing PBX project (or any collection of suitable files)

  1. Create a new empty WOLips project
  2. Copy all necessary files from the old PBX project to the new WOLips project
  3. Do Refresh, Close and Open - and you have fully functional WOLips project !) Do not use the import wizard
Decline it if the EOModeler on MacOSX asks you to add the files to the project. Otherwise your PB.project is converted to a XML file. Don't do it!

Working with PBX simultaneously. In this document, when referring to ProjectBuilder, it is the Windows and/or old Macintosh version (PBWO) that is intended – not the new, Macintosh only, version (PBX). WOLips needs to maintain either the PBWO ‘PB.project’ file or the PBX ‘*.pbproj’ file in order to have WOBuilder and EOModeler function as desired. The PBWO file is apparently much easier to maintain than the PBX file. Working in parallel with PBWO and eclipse/WOLips is possible, but is not the primary reason for maintaining the project files.

You can also have a look at the SourceForge tasks list for WOProject.
