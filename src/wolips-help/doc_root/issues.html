Bugs

Bug

Fix
Importing projects with localized components doesn't work. ?

On Windows the ~\Library\wobuild.properties is created with paths using single '\' instead of double '\\'. This will cause some WOProject Ant scripts to fail. In particular it is not possible to build WOProject and/or WOLips from source.

Edit the file, and replace all single '\' with double '\\'.
Importing projects with java code organised in packages doesn't work properly. Either the java files are not found or they may be put in the wrong directory. Adding them manually is not a big task!
In some situations eclipse freezes or throws an OutOfMemoryError. Try turning off "Build after JavaBuilder" in the WOLips preferences.

WOLips adds this to the PB.project file:

FRAMEWORKSEARCH = (
C:\Apple\Local\Library\Frameworks
);

While PBWO adds this:

FRAMEWORKSEARCH = ("C:\\Apple\\Local\\Library\\Frameworks");

The WOLips version does not work - ProjectBuilder, WOBuilder and other tools can't parse the PB.project file!

Manually fix the PB.project file each time this happens.

Issues

...things you may want to do, but can't.

Issue

Fix
Currently you can not use the ant scripts outside of Eclipse.  
The import (project) Wizards currently only work with PBWO projects. If you try it with PBX projects you may find that some file where skipped and that the eclipse .project file in incomplete.

How to import an existing PBX project (or any collection of suitable files)

  1. Create a new empty WOLips project
  2. Copy all necessary files from the old PBX project to the new WOLips project
  3. Do Refresh, Close and Open - and you have fully functional WOLips project
  4. Do not use the import wizard

EOModeler on Macintosh may convert the PB.project file to XML.

When EOModeler asks you if you want to add the file(s) to to the project - decline.
WOLips needs to maintain either the PBWO ‘PB.project’ file or the PBX ‘*.pbproj’ file in order to have WOBuilder and EOModeler function as desired. The PBWO file is apparently much easier to maintain than the PBX file. Working in parallel with PBWO and eclipse/WOLips is possible, but is not the primary reason for maintaining the project files.  
WObuilder wont display component members correctly if the import statements of its class are too tight. In Windows > Preferences > Java > Organize Imports set "Number of imports needed for .*" to 2.
