Bug

Fix

693046

When you add local framework to using PBWO this is also added to the PB.project file:

FRAMEWORKSEARCH = ("C:\\Apple\\Local\\Library\\Frameworks");

If/when you do the same thing using WOLips this does not happen.

In addition WOLips keeps transforming the section to:

FRAMEWORKSEARCH = (
C:\Apple\Local\Library\Frameworks
);

This does not work! PBWO, EOModeler and WOBuilder all stop working.

Use PBWO to add local frameworks, manually fix the PB.project file each time WOLips screws it up.

695129

Importing projects with java code organised in packages doesn't work properly. Either the java files are not found or they may be put in the wrong directory.

 Adding them manually is not a big task!

Issue

Fix

695152

The import (project) Wizards currently only work with PBWO projects. If you try it with PBX projects you may find that some file where skipped and that the Eclipse .project file in incomplete.

How to import an existing PBX project (or any collection of suitable files)

  1. Create a new empty WOLips project
  2. Copy all necessary files from the old PBX project to the new WOLips project
  3. Do Refresh, Close and Open - and you have fully functional WOLips project
  4. Do not use the import wizard

695157

EOModeler on Macintosh may convert the PB.project file to XML.

When EOModeler asks you if you want to add the file(s) to to the project - decline.

Bug / Issue

Fix
On Microsoft Windows 2000, a WebObjects 5.1 (and 5.0) application linked to several frameworks may not open or "launch". The problem is that the cmd.exe in Win2k can't handle commands longer than 2000 characters.

You have 4 alternatives:

  1. Switch to WO 5.2
  2. Switch to WinXP*
  3. Use just the cmd.exe from WinXP*
  4. Use WOLauncher.exe
*) You must also modify the MAXLEN variable of the application start script to allow commands longer than 2000 characters.
WOBuilder wont display component members correctly if the import statements of its class are "too tight". Eclipse has a tendency keep them very tight. In Windows > Preferences > Java > Organize Imports set "Number of imports needed for .*" to 2. This should be enough for most situations.
Excessive use of Ant (internally) could cause Elipse to freeze and/or throw an OutOfMemoryError. This is a problem with the Ant plug-in currently shipping with Eclipse. Try turning off "Build after JavaBuilder" in the WOLips preferences.
