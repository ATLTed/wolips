Getting Started

Preparations

You need a computer with a complete and working WebObjects 5.1/5.2 development installation (Setup WebObjects on Windows). WOLips versions prior to 0.9.2 required that you launched Eclipse using Java 1.3.1. WOLips 0.9.2 works with both Java 1.3.1 and 1.4.1, but require that you use Eclipse 2.1. At the time of writing the Eclipse “Latest Release” is still version 2.0.2. Both the “Current Stable Build” and “Current Integration Build” are however version 2.1 – use one of these. In addition, there is no “Latest Release” Macintosh version.

WOLips is mainly developed using Eclipse on Macintosh – so it definitely should work on that platform. The author of this document has, however, had some problems. If you have access to both Macintosh and Windows computers, the author currently recommends you use a Windows computer to evaluate WOLips (because of Eclipse).

If you haven’t already, download Eclipse – unpack it and move the Eclipse folder to C:\Eclipse (From now on the author assumes this is where you put it). Inside this folder you find the executable as well as the C:\eclipse\features, C:\eclipse\plugins and C:\eclipse\workspace folders.

(On Macintosh you need to Show Package Contents on the Eclipse application.)

Get to know Eclipse! Bring yourself to a level where you can create projects and run/debug simple applications before continuing. The built-in help – Help > Help Contents – contain several tutorials. Make certain you understand the concept of perspectives and views, and have a look at the project build path settings.

If you’re used to working with CVS then set that up as well, and while you’re at it, why not set up junit and write a couple of tests.

Your Very First WOLips Application

It is very easy to create and run a basic WO application using WOLips. Here’s how to do it:

  1. Open Windows > Preferences – WOLips and make sure Build after JavaBuilder is checked. This ensures that ready-to-deploy frameworks and applications are build as soon as the project is built, and as always with Eclipse, the project is built as soon as (and every time) you save changes. Turning this option on is only necessary when you actually want to build a framework or application. It is not necessary while developing.
  2. Select the menu option File > New > Other...
  3. Select WOLips WO 5.1 and 5.2 and then Application – press next.
  4. Name the project HelloWOrld – press Finish.
  5. Open the Navigator view and expand the HelloWOrld node. (As you just learned there is now a WOLips perspective and several new views. This description is based on the ordinary Navigator view.) Inside you’ll find four folders, and a number of files. They are:
    • ‘bin’ contains all the *.class files.
    • ‘src’ contains all the *.java files.
    • ‘HelloWOrld.woa’ is the executable application. Among other things this folder contains the application start script ‘HelloWOrld.cmd’. If this folder doesn’t exist two things could be wrong. 1) The Build after JavaBuild option (from step 1) isn’t on. 2) The file ~\Library\wobuild.properties (from Installing WOLips) does not exist or is incorrect. If any of these things are wrong you need to correct them and then rebuild the project (Project > Rebuild Project).
    • ‘Main.wo’ contain the *.html, *.wod and *.woo files for the Main component – as expected. The *.api file is stored directly under the project root (as in ProjectBuilder) and the *.java file is stored in the src folder (as we saw before). If/when you add more components; their files will be stored accordingly.
    • ‘.classpath’ and ‘.project’ are Eclipse project files.
    • ‘build.properties’ and ‘build.xml’ are Ant/WOProject files.
    • ‘CustomInfo.plist’, ‘Makefile’, ‘Makefile.postamble’, ‘Makefile.preamble’ and ‘PB.project’ are ProjectBuilder files.
    • ‘Properties’ is a WOLips specific file.
    • ‘WOAfile.icns’ is the application icon set.
  6. Double-click ‘PB.project’ and ProjectBuilder opens... (Return to Eclipse.)
  7. Double-click ‘Main.wod’ and WOBuilder opens... (Modify it to be a simple HelloWOrld  component.)
  8. Double-click ‘HelloWOrld.cmd’ and the application is launched...

That’s it!
