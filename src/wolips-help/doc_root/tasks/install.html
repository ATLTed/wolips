Installing WOLips

What you need

  • A computer with a complete and working WebObjects 5.1/5.2 development installation (Setup WebObjects on Windows).
  • eclipse 2.1.0 or above. At the time of writing the eclipse "Latest Release" is still version 2.0.2. Both the "Current Stable Build" and "Current Integration Build" are however version 2.1 - use one of these. In addition, there is no "Latest Release" Macintosh version.
  • JDK 1.3.1 or above - 1.4.1 is ok.
  • If you're on Macintosh you must use at least OS X 10.2.

Steps

  1. Quit eclipse if you have it running.
  2. If you have any previous version of WOLips installed then uninstall that first.
  3. Download the latest WOLips release from SourceForge.
  4. Unpack the downloaded file. You should end up with a folder named something like WOLips_x.y.z. This folder should in turn contain two folders: features and plugins.
  5. Copy whatever you find in the features and plugins folders to C:\eclipse\features and C:\eclipse\plugins respectively.
  6. Restart eclipse. (On the Mac it's normal that you have to launch Eclipse.app twice after changes to the plugins and features.) When you start eclipse with a new or updated feature you have to activate it with the Update Manager. It should appear automatically when eclipse is started. If not; you’ll find it under Help > Software Updates > Update Manager.
  7. Please reset the WOLips perspective after an update.
  8. Replace "${user.home}/wobuild.properties" with "${user.home}/Library/wobuild.properties" in your build.xml files.
