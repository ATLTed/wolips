Installing WOLips

What you need

  • Eclipse 2.1.
  • A complete and working WebObjects 5.1/5.2 Development installation. (Setup WebObjects on Windows)
  • JDK 1.3.1 or above - 1.4.1 is ok.
  • If you're on Macintosh you must use at least OS X 10.2.

Steps

  1. Quit Eclipse if you have it running.
  2. If you have any previous version of WOLips installed then uninstall that first.
  3. Download the latest WOLips release from SourceForge. *
  4. Unpack the downloaded file. You should end up with a folder named something like WOLips_x.y.z. This folder should in turn contain three folders: features, plugins and optional.
  5. Copy whatever you find in the features and plugins folders to C:\eclipse\features and C:\eclipse\plugins respectively. (The paths may be different on your system.)
  6. Read the README.html file in the folder named optional. There may be additional plug-ins or featues to install.
  7. Restart Eclipse. (On Macintosh it's 'normal' that you have to launch Eclipse twice after changes to the plugins and features.) When you start Eclipse with a new or updated feature you have to activate it with the Update Manager. It should appear automatically when Eclipse is started. If not; you’ll find it under Help > Software Updates > Update Manager.
  8. The WOLips perspective should be reset after updates.
  9. Read the What's New section of the WOLips on-line documentation. It may contain information regarding necessary actions with this particular version.

*) It is also possible, and not difficult, to build from source.
