Profiling WOLips applications

WOLips integrates with jMechanic. If the jMechanic plugin is present when you install or update WOLips (it's not enough to install jMechanic and restart eclipse if you already had WOLips installed) you get an additional launch configuration - 'Profiled WOApplication'. This configuration can be used just like the ordinary 'WOApplication' launch cinfiguration. The 'Profiled WOApplication' configuration simply has an additional tab for profiling. Profiling only occurs when you debug, not when you run.

If jMechanic is not present when you install or update WOLips there will be a line in the .log file saying that the plugin org.objectstyle.wolips.profiling could not be loaded.

In the future there will be additional views for profiling WO specific stuff.

All questions about jMechanic should go to jMechanic. The hompage is: jmechanic.sourceforge.net.
